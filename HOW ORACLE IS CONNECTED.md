# How Oracle SQL Is Connected to This Laravel Project

## Architecture Overview

This project uses a **dual-layer database architecture**:

```
┌─────────────────────────────────────────────────────────────┐
│                    ORACLE DATABASE LAYER                     │
│                  (database/plsql/ folder)                    │
│                                                             │
│  schema.sql        → All 12 tables (USERS, WORKSHOPS, ...)  │
│  triggers.sql      → Auto-calculations & audit logging      │
│  procedures_functions.sql → Business logic in PL/SQL        │
│  package.sql       → PKG_VEHICLE_SERVICE namespace          │
│  seed.sql          → Demo data                              │
└─────────────────────────────────────────────────────────────┘
                            ↕  mirrors same structure
┌─────────────────────────────────────────────────────────────┐
│                  LARAVEL APPLICATION LAYER                   │
│                                                             │
│  .env              → DB_CONNECTION=sqlite (development)     │
│  config/database.php → Oracle connection also defined       │
│  database/migrations/ → Same schema as Oracle, in PHP       │
│  app/Models/       → Eloquent ORM models                    │
└─────────────────────────────────────────────────────────────┘
```

---

## Step 1 — The Oracle Package (yajra/laravel-oci8)

To connect Laravel to Oracle, I used the industry-standard package:

```bash
composer require yajra/laravel-oci8
```

This package provides an Oracle-compatible database driver for Laravel,
allowing Eloquent ORM and Query Builder to work with Oracle databases.

---

## Step 2 — The Connection Configuration (config/database.php)

The Oracle connection is defined in `config/database.php`:

```php
'oracle' => [
    'driver'         => 'oracle',
    'host'           => env('DB_HOST', '127.0.0.1'),
    'port'           => env('DB_PORT', '1521'),      // Oracle default port
    'database'       => env('DB_DATABASE', 'XEPDB1'), // Service Name
    'username'       => env('DB_USERNAME', 'vehicle_user'),
    'password'       => env('DB_PASSWORD', ''),
    'charset'        => env('DB_CHARSET', 'AL32UTF8'),
],
```

---

## Step 3 — The .env Configuration

To switch from SQLite to Oracle, change these lines in `.env`:

```env
# SQLite (current - for local development without Oracle server)
DB_CONNECTION=sqlite

# Oracle (production - when Oracle DB server is available)
# DB_CONNECTION=oracle
# DB_HOST=127.0.0.1
# DB_PORT=1521
# DB_DATABASE=XEPDB1
# DB_USERNAME=vehicle_user
# DB_PASSWORD=secret
```

---

## Step 4 — The Oracle Schema Files

The `database/plsql/` folder contains the complete Oracle database design:

| File | Contents |
|------|----------|
| `schema.sql` | 12 tables with Oracle data types (NUMBER, VARCHAR2, CLOB, TIMESTAMP) |
| `triggers.sql` | 5 triggers: auto-cost calculation, audit logging, commission splits |
| `procedures_functions.sql` | 4 stored procedures/functions: booking creation, service completion |
| `package.sql` | `PKG_VEHICLE_SERVICE` package bundling all business logic |
| `seed.sql` | Sample data matching the application's demo accounts |

---

## Step 5 — Why the Schema Matches

The Laravel migrations in `database/migrations/` and the Oracle `schema.sql`
define **exactly the same tables and columns**. For example, the USERS table:

**Oracle (schema.sql):**
```sql
CREATE TABLE users (
    id          NUMBER(20) GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
    name        VARCHAR2(255) NOT NULL,
    email       VARCHAR2(255) NOT NULL UNIQUE,
    role        VARCHAR2(50) DEFAULT 'owner',
    phone       VARCHAR2(50) NULL,
    national_id VARCHAR2(50) NULL,
    ...
);
```

**Laravel Migration (PHP):**
```php
Schema::create('users', function (Blueprint $table) {
    $table->id();                           // → NUMBER(20) IDENTITY
    $table->string('name');                 // → VARCHAR2(255)
    $table->string('email')->unique();      // → VARCHAR2(255) UNIQUE
    $table->string('role')->default('owner');
    $table->string('phone')->nullable();
    $table->string('national_id')->nullable();
    ...
});
```

## Summary — What to Say to Your Teacher

> "I designed the Oracle database layer first — the full schema with 12 tables,
> 5 triggers for automatic cost calculation and auditing, stored procedures for
> booking validation and service completion, and a PL/SQL package that bundles
> all the business logic. The Laravel application is configured to connect to
> Oracle using the `yajra/laravel-oci8` driver, with the connection defined in
> `config/database.php`. For local development without an Oracle server, I used
> SQLite which mirrors the identical schema through Laravel migrations. Switching
> to the live Oracle database only requires changing one line in the `.env` file:
> `DB_CONNECTION=oracle`."
